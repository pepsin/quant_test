## 一、这是什么？一句话总结

这是一个运行在**聚宽（JoinQuant）**平台上的**ETF 轮动策略**。它的核心思想很简单：

> **每天从 40 只 ETF 中，选出近期"涨得最稳"的 1 只买入持有；如果都不符合条件，就躲进货币基金保命。**

所谓"轮动"，就是资金在不同资产类别之间流动——今天买黄金，明天可能切到纳斯达克，后天可能切到债券。策略通过数学模型自动完成这种切换。

**注意买卖顺序**：先卖后买（间隔 1 分钟），这样可以释放现金。

## 三、核心金融与数学概念（从零开始）

### 3.1 动量（Momentum）—— 策略的灵魂

**直觉**：物理学中的动量是 `mv`，金融中的动量是"涨惯了的东西倾向于继续涨"。这是量化投资中最经典的异象之一（Momentum Anomaly），甚至获得了诺贝尔奖（行为金融学解释）。

**本策略的动量定义**：不是简单的"25 天涨了多少"，而是**对数价格的加权线性回归斜率**。

```python
recent = price_series[-(lookback_days + 1):]  # 最近 26 个收盘价
y = np.log(recent)                            # 取对数，把乘法变加法
x = np.arange(len(y))                         # [0, 1, 2, ..., 25]
weights = np.linspace(1, 2, len(y))           # 越新的数据权重越大
slope, intercept = np.polyfit(x, y, 1, w=weights)
annualized_returns = math.exp(slope * 250) - 1
```

#### 为什么要取对数？

金融收益率的本质是乘法：今天 100 → 明天 110 → 后天 121。取对数后，`log(121) - log(100) = log(1.21)`，收益率变成了**可加的时间序列**，这才符合线性回归的前提假设。

#### 年化收益率怎么来的？

`slope` 是每天的对数收益率（近似）。一年约 250 个交易日，所以 `slope * 250` 是一年的对数收益，再 `exp()` 回来变成真实收益率。

公式推导：
```
若 price(t) = price(0) * e^(slope * t)
则 1 年后：price(250) / price(0) = e^(slope * 250)
年化收益率 = e^(slope * 250) - 1
```

#### 为什么要加权？

`weights = np.linspace(1, 2, len(y))` 表示最近一天权重是 2，最早一天权重是 1。逻辑是：**近期的价格行为比远古时期更能代表未来**。

### 3.2 R²（决定系数）—— "稳不稳"的度量

动量大但不代表值得买——可能是一根大阳线拉起来的，随时会崩。R² 衡量的是**价格走势的"线性程度"**。

```python
ss_res = np.sum(weights * (y - (slope * x + intercept)) ** 2)  # 残差平方和
ss_tot = np.sum(weights * (y - np.mean(y)) ** 2)               # 总平方和
r_squared = 1 - ss_res / ss_tot
```

- **R² = 1**：完美直线，涨跌非常有规律
- **R² = 0**：完全随机，毫无趋势
- **R² < 0**：比随机还差（拟合线比平均值还差）

**最终得分 = 年化收益率 × R²**

这个设计很巧妙：它惩罚了"暴涨暴跌型"选手，奖励"稳步上涨型"选手。一个年化 100% 但 R²=0.3 的 ETF，得分不如年化 50% 但 R²=0.8 的 ETF。

### 3.3 拉普拉斯滤波器（Laplace Filter）—— 趋势期的灵敏探头

```python
def laplace_filter(price, s=0.05):
    alpha = 1 - np.exp(-s)      # ≈ 0.0488
    L = np.zeros(len(price))
    L[0] = price[0]
    for t in range(1, len(price)):
        L[t] = alpha * price[t] + (1 - alpha) * L[t - 1]
    return L
```

这其实就是**指数移动平均（EMA）**的另一种写法。

- `alpha` 越小，平滑效果越强，对价格越不敏感
- `alpha` 越大，越贴近最新价格

**使用方式**：当前价格必须大于滤波值，且滤波值的斜率（最后两点的差）大于阈值 0.001。这保证了**价格在上行通道中，且速度足够快**。

### 3.4 高斯滤波器（Gaussian Filter）—— 震荡期的平滑卫士

```python
def gaussian_filter_last_two(price, sigma=1.2):
    idx = np.arange(n)
    weights = np.exp(-((idx + 1) ** 2) / (2 * sigma ** 2))[::-1]
    weights /= np.sum(weights)
    g = np.sum(price * weights)
```

与拉普拉斯的区别：
- **拉普拉斯**：指数衰减权重，像一个"leaky integrator"（有泄漏的积分器）
- **高斯**：钟形曲线权重，越远的数据衰减得**更快、更剧烈**

结果：高斯滤波器更平滑，能过滤掉更多噪音。在震荡市中，很多假突破会被高斯滤波器拦下来，策略因此保持空仓或切换到低风险资产。

### 3.5 RSI（相对强弱指标）—— 震荡期触发器

RSI 是技术分析的经典指标，衡量买卖双方的力量对比：

```python
deltas = np.diff(close)            # 每日涨跌
gains = np.where(deltas > 0, deltas, 0)    # 只保留涨幅
losses = np.where(deltas < 0, -deltas, 0)  # 只保留跌幅（取正）
avg_gain = np.mean(gains[-period:])         # 最近 14 天平均涨幅
avg_loss = np.mean(losses[-period:])        # 最近 14 天平均跌幅
rs = avg_gain / avg_loss
rsi = 100 - (100 / (1 + rs))
```

- RSI > 70：超买，可能回调
- RSI < 30：超卖，可能反弹

**本策略的用法**：RSI 从 >75 跌落到 <60，视为"市场从狂热中冷却"，是进入震荡期的信号之一。

### 3.6 乖离率（Bias）—— 另一个震荡期触发器

```python
ma = np.mean(close_series[-20:])     # 20 日均线
bias = (current_price - ma) / ma      # 价格偏离均线的百分比
```

当股价远高于均线（比如 >10%），说明市场情绪过热，均值回归的概率增大。策略此时切到高斯滤波器，降低敏感度，防止追高被套。

### 3.7 溢价率—— QDII/LOF 的特有风险

ETF 有两个价格：
- **净值（IOPV）**：基金实际持有的资产价值
- **交易价格**：二级市场买卖价格

```python
premium_rate = (price - net_value) / net_value
```

对于跨境 ETF（如纳指 ETF、原油 ETF），由于外汇额度限制或时差，经常会出现大幅溢价。比如 2020 年原油基金溢价 100%+，此时买入等于直接亏损。策略设置溢价率 >20% 直接排除。

### 3.8 盈利保护——移动止盈

不是止损（亏多少卖），而是**移动止盈**：从最高点回撤 5% 就卖。

```python
max_high = hist['high'].max()        # 最近 N 日最高价
current_price <= max_high * 0.95     # 回撤超过 5%，触发卖出
```

这保护了已经到手的利润，避免坐过山车。

## 四、代码逐模块详解

### 4.1 初始化模块 `initialize()`

#### ETF 池设计

策略配置了两套 ETF 池：

**大宗商品**（避险/通胀）：黄金、白银、豆粕、原油、有色、能源化工
**国际权益**（分散 A 股风险）：纳指、标普、日经、德国、法国、东南亚
**香港市场**：恒生科技、中概互联、港股红利
**A 股指数**：沪深 300、中证 500、创业板、科创板、中证 1000、中证 2000、A500
**风格因子**：红利低波、价值、自由现金流、创业板成长
**债券**：可转债、国债、城投债

**设计思想**：全覆盖不同经济周期下的优势资产。经济衰退买黄金/债券，复苏买周期股，过热买大宗商品，滞胀买现金/防御。

#### 参数表

| 参数 | 值 | 含义 |
|------|-----|------|
| `lookback_days` | 25 | 动量计算周期（约一个月） |
| `holdings_num` | 1 | 只持有 1 只 ETF（集中持仓） |
| `defensive_etf` | 511880 | 银华日利（货币基金 ETF） |
| `profit_protection_threshold` | 0.05 | 盈利保护回撤 5% |
| `loss` | 0.97 | 近 3 日任意一天跌 >3% 直接排除 |
| `premium_threshold` | 0.20 | 溢价率 >20% 排除 |
| `volume_threshold` | 2 | 成交量是 5 日均量的 2 倍以上视为"放量" |

### 4.2 打分与过滤系统 `calculate_momentum_metrics()`

这是策略最复杂的函数，一个 ETF 要经过**七层筛选**才能入选：

```
第一层：停牌过滤
    ↓
第二层：盈利保护检查（若持仓已回撤 5%，排除买入/触发卖出）
    ↓
第三层：溢价率过滤（跨境 ETF 溢价过高排除）
    ↓
第四层：成交量过滤（涨太多+放量，疑似拉高出货，排除）
    ↓
第五层：短期动量过滤（近 10 天涨幅 <0，排除）
    ↓
第六层：长期动量计算（打分 = 年化收益 × R²）
    ↓
第七层：近 3 日单日跌幅过滤（任意一天跌 >3%，排除）
    ↓
第八层：动态滤波器（正常期拉普拉斯，震荡期高斯）
    ↓
入选！
```

**为什么近 3 日跌幅过滤放在打分之后？**

因为这是一个"一票否决"条件。即使动量得分再高，只要最近有暴跌迹象，直接排除。放在后面是为了避免无谓计算——前面的过滤已经筛掉大部分了。

### 4.3 震荡期机制（策略核心创新）

这是本策略区别于普通 ETF 轮动的地方。市场不是永远有趋势的——70% 的时间在震荡。用趋势跟踪的方法做震荡市，会反复打脸（买就跌，卖就涨）。

#### 状态机设计

```
              ┌─────────────────────────────────────┐
              │                                     │
              ▼                                     │
   ┌──────────────────┐         退出条件满足       │
   │    正常期        │◄───────────────────────────┘
   │ (拉普拉斯滤波器)  │
   └────────┬─────────┘
            │ 进入条件满足
            ▼
   ┌──────────────────┐
   │    震荡期        │
   │ (高斯滤波器)      │
   └──────────────────┘
```

#### 进入震荡期的条件（任一触发）

1. **乖离率过大**：沪深 300 价格偏离 20 日均线 >10%
2. **RSI 超买回落**：RSI 从 >75 跌破 60
3. **盈利保护触发止损**（可选）：持仓触发了移动止盈

#### 退出震荡期的条件（任一触发）

1. **从低点反弹**：从近 20 日最低点上涨 ≥3%
2. **企稳信号**：回撤 <3% + 至少 2 个企稳指标（站上均线、价格回升、回撤收窄、RSI 回升）+ 连续 ≥2 天
3. **强制到期**：震荡期最长 15 个交易日，强制退出

#### 冷却机制

切换后有 2 个交易日的冷却期，防止在边界条件附近反复横跳（chatter）。这是状态机设计中的经典**防抖（debounce）**机制。

### 4.4 交易执行

#### 卖出 `etf_sell_trade()`

- 获取今日排名
- 前 `holdings_num`（1）名是目标持仓
- 当前持仓不在目标列表中的，全部卖出
- 如果没有目标 ETF 且防御 ETF 可用，把防御 ETF 设为目标（这样不会卖出防御仓位）

#### 买入 `etf_buy_trade()`

- 获取今日排名
- 从第 1 名开始往下遍历，凑够 `holdings_num` 只
- 如果凑不够，进入防御模式（买入货币基金）
- **等权分配**：每只目标 ETF 分配 `总资金 / 数量`
- **5% 容差调仓**：如果当前市值与目标市值偏差 <5%，不操作（降低交易摩擦）

#### 智能下单 `smart_order_target_value()`

这个函数处理了中国 A 股的很多实操细节：

1. **停牌检查**：跳过
2. **涨跌停检查**：涨停不买，跌停不卖
3. **价格零值检查**：防止除零
4. **100 股整数倍**：A 股交易单位
5. **最小交易金额**：<5000 元不交易（避免手续费占比过高）
6. **T+1 处理**：当天买入的不能卖出，`closeable_amount` 是可卖出数量

### 4.5 缓存机制 `get_cached_rankings()`

一天内会多次调用排名（卖出前、买入前、盈利保护检查可能也需要），加入日期缓存避免重复计算：

```python
if g.rankings_cache['date'] != today:
    重新计算
else:
    用缓存
```

注意：13:55 震荡期检查后会**强制清空缓存**，因为滤波器可能切换，排名需要重新算。

## 五、一天内的完整数据流

以某个普通交易日为例：

```
09:10 开盘检查
  └─> 打印当前持仓，确认状态

11:00 盈利保护检查（假设配置了这个时间点）
  └─> 遍历持仓，若某只从高点回撤 5%，立即卖出
  └─> 若启用了止损触发震荡期，记录信号

13:10 卖出阶段
  └─> 计算 ETF 排名（缓存）
  └─> 卖出不在目标列表的持仓

13:11 买入阶段
  └─> 读取缓存排名
  └─> 确定目标 ETF（最多 1 只）
  └─> 若有旧持仓未卖出，推迟到明天（避免资金不足）
  └─> 等权买入/调仓

13:55 震荡期检查（核心）
  └─> 先检查退出条件（若已在震荡期）
  └─> 再检查进入条件（若在正常期）
  └─> 清空排名缓存
  └─> （注意：此时卖出已过，买入已过，今天的交易基于旧状态）
  └─> 明天的交易将基于新状态

15:10 收盘重置
  └─> 统计震荡期持续天数
```

**关键观察**：震荡期检查在 13:55，但买卖在 13:10-13:11。这意味着：
- **今天的交易决策基于昨天的市场状态**
- **状态切换的影响从明天开始生效**

这是一种保守设计，避免盘中临时切换导致冲动交易。但也意味着状态切换有**半天的滞后**。

## 六、策略的数学本质（总结）

如果把策略抽象成一个数学函数：

```
目标函数：max Score(ETF_i) = AnnualizedReturn(i) × R²(i)
约束条件：
  1. Premium(i) < 20%
  2. MaxDrawdown3D(i) < 3%
  3. ShortMomentum(i) > 0
  4. VolumeSignal(i) == False  (unless return < 100%)
  5. Filter(i) == True
       where Filter = Laplace  if MarketState == Normal
                    = Gaussian if MarketState == RangeBound
  6. MarketState = f(沪深300 的 Bias, RSI, 时间)
```

这是一个**带约束的优化问题**，每次选 Top-1。

## 七、优势与潜在风险

### 7.1 设计亮点

1. **双层动量**：长期动量（25 天）选标的 + 短期动量（10 天）过滤，避免买在短期山顶
2. **趋势适应**：拉普拉斯/高斯双滤波器 + 震荡期自动识别，试图解决"趋势策略死于震荡"的千古难题
3. **多层风控**：盈利保护、溢价率、成交量、单日跌幅，层层设防
4. **防御机制**：无标的时自动切货币基金，不做强行交易

### 7.2 潜在问题（老程序员的批判视角）

1. **参数过多**：20+ 个可调参数，存在过拟合风险。回测表现好不代表未来好。
2. **状态机滞后**：震荡期 13:55 判断，但交易 13:10 执行，状态切换延迟半天到一天半。
3. **单点持仓**：`holdings_num=1` 集中度极高，一旦判断错误单次回撤会很大。
4. **高斯滤波器计算量**：每次对每只 ETF 都要做高斯卷积，虽然优化为只算最后两点，但仍有 O(n) 的权重计算。
5. **RSI 和乖离率阈值固定**：8%、75/60 这些数字是硬编码的，未随市场波动率自适应调整。
6. **佣金设置过于乐观**：`0.01%`（万 1）是理想情况，实际散户很难拿到，且未考虑印花税（虽然 ETF 免印花税）。

### 7.3 实盘注意事项

- **滑点**：ETF 流动性差异很大，小盘 ETF 的盘口很薄，`PriceRelatedSlippage(0.01%)` 可能不够
- **净值延迟**：QDII ETF 的净值 T+1 甚至 T+2 才公布，溢价率计算可能用旧数据
- **停牌风险**：跨境 ETF 因外汇额度不足可能暂停申购，导致溢价持续扩大，此时溢价率过滤反而让你错过机会（或躲过陷阱，取决于你怎么看）

## 八、如果让你重写，可以怎么改进？

（给程序员的思考方向）

1. **用向量化计算替代循环**：当前 `calculate_momentum_metrics()` 是逐个 ETF for 循环，可以用 `pandas.groupby` 或 `numba` 加速
2. **引入波动率自适应**：用 ATR（平均真实波幅）或波动率指数动态调整阈值
3. **机器学习替代硬编码规则**：用随机森林或 XGBoost 学习"震荡期"的定义，而非人工设定 RSI/Bias 阈值
4. **仓位管理**：当前是等权或全仓单吊，可以加入凯利公式或风险平价（Risk Parity）分配资金
5. **事件驱动优化**：把盈利保护从定时轮询改为条件触发，减少不必要的计算

## 九、术语速查表

| 术语 | 解释 |
|------|------|
| ETF | 交易所交易基金，像股票一样买卖的基金包 |
| LOF | 上市开放式基金，兼具场内场外交易能力 |
| QDII | 合格境内机构投资者，允许投资境外市场的基金 |
| 动量 | 涨得好的继续涨，跌得多的继续跌的趋势现象 |
| EMA | 指数移动平均，越近的数据权重越高 |
| RSI | 相对强弱指标，0-100，衡量超买超卖 |
| 乖离率 | 价格与均线的偏离百分比 |
| 溢价率 | 场内价格相对基金净值的溢价百分比 |
| R² | 决定系数，衡量拟合优度 |
| 年化收益 | 把任意周期的收益折算为一年的收益率 |
| T+1 | A 股交易制度，当天买入次日才能卖出 |
| 滑点 | 下单价格与实际成交价格的差异 |
| 回测 | 用历史数据验证策略表现 |
| 过拟合 | 参数过度适配历史数据，导致未来失效 |

> **最后一句**：这本质上是一个**规则交易系统（Rule-Based Trading System）**，用程序员的话说就是——一堆 if-else 组成的专家系统。它的威力不在于某个单独的数学公式，而在于多层规则的叠加和互补。理解每一个 if 背后的金融逻辑，你就理解了量化投资的入门之道。
