# ETF 轮动策略脚本详解

> 本文面向有编程经验但不熟悉量化/数理分析的读者。所有金融术语都会用程序员视角解释。

---

## 一、这个策略是干什么的？

**一句话版本：** 从一个 ETF 池子里，每天用算法挑出"涨势最好"的那只 ETF 持有；如果市场进入震荡状态，就自动切换到更保守的过滤算法；如果实在没有好标的，就躲进货币基金"趴着等"。

**类比：** 把它想象成一个自动赛车换手系统——赛道条件好时开快车（拉普拉斯滤波器），赛道湿滑时开慢车（高斯滤波器），赛道完全封闭时停进安全区（货币基金防御模式）。

---

## 二、什么是 ETF？

ETF（交易所交易基金）就是一篮子股票打包成的"股票"，可以像股票一样在交易所买卖。本脚本涉及的 ETF 品类：

| 类型 | 举例 |
|------|------|
| 大宗商品 | 黄金 ETF、豆粕 ETF、原油 ETF |
| 国际市场 | 纳指 ETF（美国纳斯达克）、日经 ETF（日本）|
| 香港市场 | 恒生科技、港股互联 |
| A 股指数 | 沪深 300、创业板、科创 50 |
| 债券 | 国债 ETF、城投债 ETF |
| 防御资产 | 货币基金（511880，相当于现金） |

## 三、整体架构

脚本运行在聚宽（JoinQuant）量化平台上，使用该平台的 API。整体结构如下：

```
initialize()          ← 相当于 main() 初始化入口
    │
    ├── 每日 09:10 → check_positions()         # 打印持仓日志
    ├── 每日 11:00 → profit_protection_check() # 盈利保护检查（止盈）
    ├── 每日 13:55 → check_range_bound()       # 震荡期状态判断（核心）
    ├── 每日 13:10 → etf_sell_trade()          # 卖出不符合条件的 ETF
    ├── 每日 13:11 → etf_buy_trade()           # 买入符合条件的 ETF
    └── 每日 15:10 → reset_range_bound_daily() # 收盘后重置标志
```

注意执行顺序：先判断震荡期（13:55）→ 再卖出（13:10，注意这里平台的 13:10 比 13:55 先执行，但下单在下午，实际逻辑是：震荡期状态影响下午的计算）→ 再买入（13:11）。

## 四、核心数学概念（程序员版解释）

### 4.1 动量（Momentum）

**通俗理解：** "涨势好的继续涨"这个规律。类比：一个 HTTP 请求响应时间过去一个月都很快，大概率下个月也快。

**代码实现：** 取最近 25 天的收盘价，对价格的自然对数做加权线性回归，斜率越大 = 涨势越猛。

```python
y = np.log(recent)          # 对价格取对数（放大小数差异，压缩大数差异）
x = np.arange(len(y))       # [0, 1, 2, ..., 25]
weights = np.linspace(1, 2, len(y))  # 越新的数据权重越高（1→2 线性增加）
slope, intercept = np.polyfit(x, y, 1, w=weights)  # 加权最小二乘拟合一条直线
annualized_returns = math.exp(slope * 250) - 1     # 把斜率换算成"年化收益率"
```

`* 250` 的原因：A 股一年约 250 个交易日，把每日斜率放大成年度收益。

---

### 4.2 R²（R 平方）—— 趋势稳定性

**通俗理解：** 涨得好不算，还要涨得"稳"。R² 是 0~1 之间的数，越接近 1 说明价格走势越像一条直线（趋势越稳定）；越接近 0 说明价格乱跳（噪音大）。

**类比：** 两个应用，A 每天请求量从 100 线性增长到 1000，B 在 100~1000 之间随机跳动，平均值相同。R² 能区分 A（稳定趋势）和 B（噪音）。

```python
ss_res = 实际值与拟合直线的偏差平方和（加权）
ss_tot = 实际值与均值的偏差平方和（加权）
r_squared = 1 - ss_res / ss_tot
```

**最终得分：**
```python
score = annualized_returns * r_squared
# = 年化收益率 × 趋势稳定性
# 既要涨得好，又要涨得稳
```

---

### 4.3 拉普拉斯滤波器（正常期使用）

**本质：** 就是 EMA（指数移动平均），在信号处理领域叫拉普拉斯滤波器。

```python
def laplace_filter(price, s=0.05):
    alpha = 1 - np.exp(-s)   # s=0.05 时，alpha ≈ 0.0488（很小的权重）
    L[0] = price[0]
    for t in range(1, len(price)):
        L[t] = alpha * price[t] + (1 - alpha) * L[t-1]
    # 等价于 EMA，新价格权重 ~5%，历史价格权重 ~95%
```

**判断逻辑：**
- 当前价格 > 滤波后的价格（说明在"均线上方"）
- 滤波曲线的斜率 > 0.001（说明均线在上升）

两条都满足 → 通过过滤，认为该 ETF 处于上升趋势。

**为什么叫"灵敏"：** s=0.05 时反应较快，价格变化后几天内就能体现到滤波值中，适合趋势行情跟踪。

---

### 4.4 高斯滤波器（震荡期使用）

**本质：** 用高斯（正态分布）权重对历史价格做加权平均。越近的数据权重越大，但衰减比 EMA 更平滑（钟形曲线衰减 vs 指数衰减）。

```python
# 以"距当前位置"为距离，用高斯公式算权重
weights = exp(-distance² / (2 * sigma²))
# sigma=1.2：衰减较快，只关注最近几个数据点
```

**为什么比拉普拉斯更"平滑"：** 高斯权重在距离较远时衰减更彻底，相当于用更"近视"的眼睛看价格，噪音自然被过滤更多。

**代价：** 反应更迟钝，趋势行情时可能错过早期信号。所以只在震荡期使用，震荡期需要的正是"过滤假信号"的能力。

---

### 4.5 乖离率（Bias Rate）

**通俗理解：** 价格偏离均线的百分比。类比：CPU 当前使用率 vs 过去 20 天平均使用率。

```python
ma = 最近20日收盘价均值
bias = (current_price - ma) / ma
# bias > 10%：说明价格涨得太快，已经远离均线 → 超买信号
```

**为什么是震荡期触发条件：** 价格大幅偏离均线后，往往会"均值回归"（就像橡皮筋被拉得太远会弹回来），这时候容易出现震荡或回调。

---

### 4.6 RSI（相对强弱指数）

**通俗理解：** 衡量"近期涨多了还是跌多了"的指标，值域 0~100。

```python
gains  = 过去14天中每天的涨幅（跌的天算0）
losses = 过去14天中每天的跌幅（涨的天算0）
avg_gain = mean(gains)
avg_loss = mean(losses)
rs  = avg_gain / avg_loss   # 平均涨幅/平均跌幅
rsi = 100 - 100 / (1 + rs)
```

- RSI > 75：超买（涨太猛，容易回调）
- RSI < 30：超卖（跌太多，可能反弹）

**震荡期触发条件：**
```
上一天 RSI > 75（超买）
且
今天 RSI < 60（从超买跌落）
```
说明多头力量快速衰退，市场可能进入震荡。

## 五、震荡期机制（策略核心）

这是整个策略最复杂的部分，本质上是一个**状态机**：

```
         触发条件满足
正常期  ──────────────►  震荡期
        ◄──────────────
         退出条件满足
```

### 5.1 进入震荡期（任一条件触发）

| 条件 | 说明 |
|------|------|
| 乖离率 > 10% | 沪深300 价格偏离20日均线超10% |
| RSI超买回落 | RSI从>75跌破60，说明多头在撤退 |
| （可选）盈利保护止损 | 本脚本中此开关默认关闭 |

### 5.2 退出震荡期（任一条件触发）

| 条件 | 说明 |
|------|------|
| 从近期低点上涨 ≥ 3% | 市场已从底部恢复 |
| 回撤 < 3% + 多个企稳信号 + 连续2天 | 综合企稳信号 |
| 震荡期满 15 个交易日 | 强制超时退出，防止一直待在防御状态 |

### 5.3 冷却机制

切换后有 **2 个交易日冷却期**，防止"抖动"（频繁来回切换）。类比软件中的 debounce 防抖。

### 5.4 状态对日常交易的影响

```
正常期：用拉普拉斯滤波器过滤 ETF（标准较宽松，能选出更多标的）
震荡期：用高斯滤波器过滤 ETF（标准更严格，很多 ETF 会被过滤掉）
         → 更容易触发防御模式（选不出好标的 → 切换货币基金）
```

## 六、各过滤器执行流程

每次为一只 ETF 计算指标时，按顺序经过 7 道过滤：

```
ETF候选
  │
  ├─ [1] 停牌检查          → 停牌直接跳过
  ├─ [2] 盈利保护检查      → 从最近1天最高价回撤>5% → 排除（且触发卖出）
  ├─ [3] 溢价率过滤        → 场内价格 / 基金净值 > 120% → 排除（买贵了不买）
  ├─ [4] 成交量放量过滤    → 当日量 > 过去5日均量×2 且年化>100% → 排除（异常炒作）
  ├─ [5] 短期动量过滤      → 最近10天年化收益 < 0% → 排除（近期在跌）
  ├─ [6] 长期动量计算      → 计算 score = 年化收益率 × R²
  ├─ [7] 近3日单日跌幅     → 有任意一天跌超3% → 排除
  └─ [8] 动态滤波器        → 正常期用拉普拉斯，震荡期用高斯
         → 未通过滤波器 → 排除

  → 通过所有过滤 → 进入最终排名
```

## 七、溢价率（Premium Rate）

ETF 在交易所的**交易价格**和基金公司公布的**净值**之间会有差异：

```
溢价率 = (交易价格 - 基金净值) / 基金净值
```

- 溢价率 > 0：交易价比净值贵，说明市场情绪亢奋（买的人多）
- 溢价率 > 20%：说明被炒得太高，存在均值回归风险 → 本策略排除

这个过滤特别针对**跨市场 ETF**（如 QDII 类产品，跟踪海外市场，因为时差净值更新慢，容易出现溢价泡沫）。

---

## 八、盈利保护机制

**本质：** 跟踪止损（Trailing Stop Loss）的变体。

```python
max_high = 最近1天的最高价
current_price = 当前价格

if current_price <= max_high * (1 - 5%):
    # 从最近高点回撤超过5% → 触发止盈卖出
```

**与普通止损的区别：**
- 普通止损：从**买入价**跌X% 就卖
- 盈利保护：从**最近高点**回落X% 就卖（高点会随价格上涨而动态更新）

**执行时间点：** 每天 11:00 检查一次，独立于主交易逻辑。

## 九、交易执行（smart_order_target_value）

这是实际下单的函数，处理了所有"边界情况"：

| 检查项 | 说明 |
|--------|------|
| 停牌检查 | 停牌不能交易 |
| 零价格检查 | 防止除零错误 |
| 100股取整 | A股ETF最小交易单位是100份 |
| 涨停不买 | 涨停时挂单成交概率很低 |
| 跌停不卖 | 跌停时也卖不出去 |
| 最小交易金额5000元 | 防止产生过小的碎股仓位 |
| T+1 检查 | A股当天买入不能当天卖出 |

## 十一、手续费与交易成本

### 11.1 脚本设定值

```python
set_slippage(PriceRelatedSlippage(0.0001), type="fund")   # 滑点：万分之一
set_order_cost(
    OrderCost(
        open_tax=0,                  # 买入印花税：0
        close_tax=0,                 # 卖出印花税：0
        open_commission=0.0001,      # 买入佣金：万分之一
        close_commission=0.0001,     # 卖出佣金：万分之一
        close_today_commission=0,    # 当日平仓额外费用：0
        min_commission=5,            # 每笔最低佣金：5元
    ),
    type="fund",
)
```

| 参数 | 值 | 含义 |
|------|----|------|
| `open_tax` / `close_tax` | 0 | ETF 免印花税（股票卖出是千分之一），✅ 符合现实 |
| `open_commission` / `close_commission` | 万分之一 | 买卖佣金，主流券商 ETF 约万分之一～万分之三，脚本偏乐观但在合理范围内 |
| `close_today_commission` | 0 | ETF 本身 T+1 当天卖不了，此项实际无意义 |
| `min_commission` | 5 元 | 每笔最低佣金 5 元，**关键限制项** |
| 滑点 | 万分之一 | 模拟挂单价与实际成交价的偏差，大 ETF 流动性好时合理，小众 ETF 实际可能更高 |

### 11.2 综合成本计算

**单次买入或卖出：**
```
佣金：成交金额 × 0.01%
滑点：成交金额 × 0.01%
合计：成交金额 × 0.02%（万分之二）
```

**一个完整的买入 + 卖出来回：**
```
总成本 ≈ 成交金额 × 0.04%（万分之四）
```

### 11.3 最低佣金 5 元的实际影响

脚本中 `g.min_money = 5000`，单笔最小交易 5000 元。此时佣金会被 5 元下限"托底"：

```
5,000 元 × 0.01% = 0.5 元  <  最低佣金 5 元
→ 实际佣金 = 5 元
→ 真实费率 = 5 / 5,000 = 0.1%（万分之十，比设定高 10 倍）

50,000 元 × 0.01% = 5 元   =  最低佣金 5 元
→ 50,000 元是盈亏平衡点

500,000 元 × 0.01% = 50 元 >  最低佣金 5 元
→ 真实费率 = 万分之一，与设定一致
```

**结论：资金越小，实际手续费率越高，回测结果越偏乐观。** 资金量至少在 5 万元以上，手续费才能接近回测中的设定值。

## 十三、整体运行流程（一天完整时间轴）

```
09:10  check_positions()          打印当前持仓（仅日志，不操作）
         │
11:00  profit_protection_check()  检查所有持仓是否从高点回撤>5%
         │                        → 是 → 立刻卖出
         │
13:55  check_range_bound()        判断沪深300状态
         │                        → 先检查是否退出震荡期
         │                        → 再检查是否进入震荡期
         │                        → 清除排名缓存（让后续重新计算）
         │
13:10  etf_sell_trade()           重新计算所有ETF排名
         │                        → 不在目标列表的持仓 → 卖出
         │
13:11  etf_buy_trade()            按新排名买入目标ETF
         │                        → 无好标的 → 买货币基金
         │
15:10  reset_range_bound_daily()  统计震荡期天数，重置每日标志
```

## 十四、策略逻辑总结

```
是否处于震荡期？
    ↓ 否（拉普拉斯，灵敏）       ↓ 是（高斯，保守）
筛选上升趋势ETF（宽松）      筛选上升趋势ETF（严格）
    ↓                               ↓
有符合条件的ETF？            有符合条件的ETF？
    ↓ 是                            ↓ 是
按 score 排名取第1名         按 score 排名取第1名
(score = 年化收益 × R²)      (score = 年化收益 × R²)
    ↓ 否                            ↓ 否
进入防御：买货币基金         进入防御：买货币基金
```

**核心思想：** 趋势跟踪 + 市场状态自适应 + 多重风控过滤。不预测市场方向，只跟随已经出现的强势信号，同时用多层过滤避免被假信号欺骗。

## 十五、策略综合评价

### 15.1 优点

**工程质量扎实**
代码结构清晰，有完整的异常处理、日志、边界检查（T+1、涨跌停、停牌、最小金额），不是随手写的玩具代码。

**风控层次丰富**
7 道过滤关卡 + 盈利保护 + 震荡期防御 + 货币基金兜底，整个风控体系是完整的，不是只会追涨。

**自适应机制是亮点**
正常期/震荡期自动切换这个设计思路很好，本质上是在"市场状态识别"上做了努力，比纯粹的动量策略进步了一步。

---

### 15.2 缺点与风险

**1. 单只持仓，风险极度集中**

`holdings_num = 1`，全仓押一只 ETF。任何一次判断失误，就是全仓亏损。这不是"量化"，更像是"算法择时的单票操作"。

**2. 动量策略的固有死穴：趋势反转**

动量策略在趋势行情里很好用，但在急速反转（比如黑天鹅事件、政策突变）时，信号滞后，往往是"涨得最猛的那只跌得也最猛"。盈利保护的回看只有 1 天，几乎等于"当日最高价"，极容易被日内波动误触发。

**3. 震荡期判断基准与 ETF 池错位**

震荡期判断盯的是**沪深 300**，但 ETF 池里有大量非 A 股资产：纳指、黄金、原油、豆粕、日经……这些资产跟沪深 300 的相关性很低。沪深 300 震荡，纳指可能还在涨，但策略会对所有标的都切换到保守模式，逻辑上有点错位。

**4. 参数非常多，过拟合风险高**

光是震荡期相关的参数就有十几个：乖离率阈值、RSI 超买线、RSI 回落线、低点涨幅阈值、回撤阈值、最大震荡天数、冷却天数……这么多参数在历史数据上回测，很容易"调出"漂亮的曲线，但换一段时间可能完全失效。这是量化回测里最常见的陷阱。

**5. 来源是社区分享文章**

代码头部明确注明克隆自聚宽社区文章。这类文章的回测结果通常经过了（有意无意的）参数优化，**样本内表现好不代表样本外也好**。作者没有义务公布真实的样本外验证结果。

**6. 部分 ETF 流动性存疑**

池子里有些标的（比如 513290 纳指生物、563300 中证 2000、513310 中韩半导体）日成交量较小，实际滑点会远超设定的万分之一，回测成本被低估。

### 15.4 如果要改进，优先做这几件事

1. **样本外回测**：用作者没用过的时间段（比如最近 1 年）单独验证，不要只看作者贴出的历史曲线
2. **参数敏感性测试**：把关键参数各自 ±20%，看结果是否还稳定；如果参数稍微一动结果就大变，说明过拟合严重
3. **增加持仓数量**：`holdings_num` 改成 2～3，降低单仓集中风险
4. **修正震荡期基准**：考虑用 ETF 池自身的平均表现来判断震荡，而不是只盯沪深 300（对海外 ETF 不公平）
